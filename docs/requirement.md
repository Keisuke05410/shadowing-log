# Shadowing Log - 要件定義書

## 概要

英語シャドーイング練習の記録・可視化アプリ。
日々の練習を手軽に記録し、継続のモチベーションを維持するための可視化を提供する。

- **利用者**: 個人（開発者本人）
- **動作環境**: ローカル（ブラウザ上で動作、サーバー不要）
- **対象ブラウザ**: Chrome最新版
- **対象デバイス**: PC優先。記録フォームはスマートフォンでも操作可能なサイズにする

---

## 用語定義

| 用語       | 定義                                             |
| ---------- | ------------------------------------------------ |
| セッション | 1回のシャドーイング練習記録（PracticeSession型） |
| 教材       | 練習対象コンテンツ（Material型）                 |
| ストリーク | 連続して練習した日数                             |

---

## やらないこと

- サーバーサイド / API
- ユーザー認証
- 他人との比較機能
- 音声の録音・分析
- 過度に詳細な統計画面
- 複数デバイス間のデータ同期
- プッシュ通知・リマインダー
- 練習中のタイマー機能
- 複数タブでの同時操作のサポート

## MVP（Phase 1）機能

### 1. 練習記録

1画面・最小摩擦で記録を完了できること。

教材が0件の場合（全削除後を含む）、記録画面はフォームの代わりに「教材を登録してから記録しましょう」メッセージと設定画面へのリンクを表示する。

#### 教材の選択・登録

- 最後に使った教材をデフォルト選択した状態でフォームを開く（`sessions` の最新レコードから `materialId` を取得）
- 直前のセッションの教材が削除済みの場合、教材リストの先頭をデフォルト選択する
- 変更したい場合のみドロップダウンで別の教材を選択
- 教材が1件以上ある場合も、ドロップダウンの末尾に「+ 新規登録」選択肢を表示する
- 教材がゼロの場合は直接「新規登録」フォームをインライン展開

新規登録時:

| 項目       | 型     | 入力方法                 | 必須 | 制約                 |
| ---------- | ------ | ------------------------ | ---- | -------------------- |
| 教材名     | string | テキスト入力             | ○    | 1〜100文字           |
| 教材の長さ | number | 分:秒 で入力（例: 2:30） | ○    | 1秒〜3600秒（1時間） |
| URL        | string | YouTube等のリンク        | -    | 最大2048文字         |

#### 練習の記録

| 項目           | 型     | 入力方法                                     | 必須 | 制約                                   |
| -------------- | ------ | -------------------------------------------- | ---- | -------------------------------------- |
| 日付           | date   | 自動入力（今日）、変更可                     | ○    | 未来日付は不可。過去の制限なし         |
| 教材           | select | デフォルトは直前の教材                       | ○    | -                                      |
| 練習時間（分） | number | プリセットボタン（5/10/15/30）+ カスタム入力 | ○    | 1〜180分                               |
| 自己評価       | 1-5    | 星アイコンをタップ                           | ○    | 初期値は未選択。保存時にバリデーション |
| メモ           | string | フリーテキスト                               | -    | 最大500文字                            |

- 同じ日に複数回の記録が可能（朝と夜など）

#### 記録の削除

- 履歴画面から削除できる（確認ダイアログあり）
- 削除確認ダイアログ：「この練習記録を削除しますか？」
- 確認ダイアログは `window.confirm()` を使用する（MVP）
- 間違えた場合は「削除 → 再登録」で対応（編集機能はPhase 2）

### 2. ダッシュボード（ホーム画面）

上から順に以下を表示：

1. **今日のステータス + 記録ボタン** - 「済 ✓（合計○分）」or 「今日のシャドーイング、始めますか？」を記録ボタンと一体化。1件でも記録があれば「済」
2. **ストリーク表示** - 現在の連続日数（大きく表示）+ 過去最長記録
3. **今週の合計** - 今週（月曜始まり）の練習合計時間を数値で表示（例: 「今週 45分」）
4. **ヒートマップ** - GitHub風コントリビューショングラフ（直近6ヶ月 = 当月を含む過去6カレンダー月）

#### ストリーク表示の詳細

- 現在の連続練習日数（`sessions` から都度計算。キャッシュしない）
- 過去最長ストリーク（同上）
- ストリーク継続条件：直近の記録日が今日または昨日であれば継続とみなす
- 表示値：記録のある連続日数（同日複数セッションは1日としてカウント）
- 断絶：前日に記録がなければ現在のストリーク = 0
- 今日未記録の場合: 前日までの連続日数を表示し、補足テキスト「今日はまだ記録がありません」を表示（グレー文字、中立トーン）
- **断絶メッセージ**: ストリーク=0 かつ過去に記録がある場合、ストリーク表示エリアにインラインで「○日間の記録がありました。また今日から始めましょう」と表示
  - ○ = 直前のストリーク日数（最後に途切れるまでの連続日数）

#### ヒートマップの詳細

- 横軸 = 週、縦軸 = 曜日
- 色の濃さ = その日の練習合計時間
- 4段階: level 0 = 未実施 / level 1 = 1〜15分 / level 2 = 16〜30分 / level 3 = 31分以上
- 日付セルをクリック → ツールチップで当日の記録サマリーを表示（件数・合計時間・教材名）
- 記録がある日のツールチップに「詳細を見る」リンク → 履歴画面の該当日にジャンプ
- 未練習の日は何も表示しない

> **技術リスク**: react-activity-calendar のツールチップカスタマイズについて、実装開始時に技術検証を行う。カスタムツールチップが不可の場合、日付クリックでモーダル表示に変更する。

#### Empty State（初回利用時）

- 「まだ練習記録がありません。最初の記録を追加しましょう！」とメッセージを表示
- 記録ボタンを画面中央に大きく配置
- ヒートマップは空の状態で表示（「ここが埋まっていく」というイメージ）

### 3. 履歴画面

- **日付グループのタイムライン形式**: 「2026年3月18日（水）」の見出しの下に、その日の記録カードを並べる
- 各カードに表示: 教材名、練習時間、自己評価（星）、メモの先頭30文字（超過分は「…」で省略）
- 削除ボタン（確認ダイアログ付き）
- デフォルトは新しい順
- **月単位の折りたたみ**: 「直近1ヶ月」= 当月のカレンダー月（例: 3月18日に閲覧した場合、3月分は展開表示、2月以前は折りたたみ）。折りたたみ見出し例:「2026年2月（18件）▼」
- 教材が削除済みの場合は「（削除済み教材）」と表示
- 記録が0件の場合：「まだ記録がありません」と表示し、記録画面への導線を配置

### 4. 設定画面

#### 教材管理

- 登録済み教材の一覧表示（名前・長さ・URL）。登録日時の降順（最新が上）
- 教材の編集（名前・長さ・URL の変更）
- 教材の削除（物理削除。紐づくセッションの教材名は「削除済み教材」と表示）
  - 削除確認ダイアログ: 「この教材に紐づく○件の記録があります。教材を削除しても記録は残ります。」
- 教材が0件の場合：「教材を追加しましょう」と表示し、追加フォームを直接表示

#### エクスポート / インポート

- **エクスポート**: `materials` と `sessions` を1つのJSONファイルとしてダウンロード。ファイル名: `shadowing-log-YYYY-MM-DD.json`
- **インポート**: JSONファイルを読み込み、既存データを**全上書き**する
  - 確認ダイアログ: 「現在のデータはすべて上書きされます。よろしいですか？」
  - バリデーション：JSONパース可能であること、`materials`と`sessions`が配列として存在すること。各レコードのフィールド型チェックは行わない（実装時判断）
  - 不正なJSON → エラー表示「ファイルの形式が正しくありません」
  - 必須フィールドの欠損 → エラー表示「データが不完全です。インポートできません」
  - インポート成功時は「データをインポートしました」トーストを表示
  - インポート後にストリーク等を再計算

## 画面構成・ナビゲーション

### 画面一覧

HashRouter による SPA。ブラウザの戻る/進むボタンが自然に機能する。

| パス          | 画面           | 説明                                   |
| ------------- | -------------- | -------------------------------------- |
| `/#/`         | ダッシュボード | ホーム画面。ストリーク・ヒートマップ等 |
| `/#/record`   | 記録フォーム   | 練習記録の新規作成                     |
| `/#/history`  | 履歴           | 過去の記録一覧・削除                   |
| `/#/settings` | 設定           | 教材管理・エクスポート/インポート      |

### ナビゲーションUI

ヘッダーにアプリ名「Shadowing Log」とナビリンクを横並びで配置:

- ダッシュボード
- 記録
- 履歴
- 設定

### 画面遷移フロー

```
ダッシュボード → [記録ボタン] → 記録フォーム → [保存] → ダッシュボード（肯定トースト表示）
ダッシュボード → [ヒートマップの日付クリック] → ツールチップ → [詳細を見る] → 履歴画面（該当日）
履歴画面 → [削除] → 確認ダイアログ → 履歴画面
設定画面 → [エクスポート] → ファイルダウンロード
設定画面 → [インポート] → 確認ダイアログ → 設定画面（再読込）
```

- **記録保存後の遷移**: ダッシュボードに戻る。肯定トーストを表示
- **復帰時のトースト**: ストリーク断絶期間後の初回記録保存時は、通常の肯定トーストの代わりに「戻ってきましたね！」トーストを表示
- ヒートマップからの日付ジャンプはreact-routerのstate渡しで実装。ページリロード時はstateが消えるため最新月を表示

## Phase 2（習慣定着後の拡張）

- **記録の編集機能**: 既存の記録を修正できるようにする
- **マイルストーン / バッジ**: 達成時にさりげなく表示
  - 時間系: 10h / 30h / 100h / 300h / 500h
  - 継続系: 7日 / 14日 / 21日 / 30日 / 60日 / 90日 / 180日 / 365日連続
- **累積グラフ**: 週ごとの練習時間を棒グラフで表示
- **週間サマリー**: 今週の練習時間の詳細、先週との比較
- **自己評価の推移グラフ**: 週/月ごとの平均自己評価
- **ストリークフリーズ**: 週1回まで休んでもストリーク維持
- **反復回数の表示**: 練習時間 ÷ 教材の長さ（教材ごとの統計とセット）

- 教材ジャンル分類と多様性の可視化

## データモデル（MVP）

```typescript
// 教材マスタ
interface Material {
  id: string; // UUID
  name: string; // 教材名（1〜100文字）
  lengthSeconds: number; // 教材の長さ（秒）、1〜3600
  url?: string; // YouTube等のURL（任意、最大2048文字）
  createdAt: string; // ISO 8601
}

// 練習記録
interface PracticeSession {
  id: string; // UUID
  date: string; // YYYY-MM-DD（ローカルタイムゾーン基準）
  materialId: string; // 教材への参照（教材削除後は孤立参照となる）
  durationMinutes: number; // 練習時間（分）、1〜180
  selfEvaluation: 1 | 2 | 3 | 4 | 5; // 自己評価
  memo?: string; // メモ（任意、最大500文字）
  createdAt: string; // ISO 8601
}
```

### タイムゾーンの扱い

日付の計算はすべてブラウザのローカルタイムゾーンに依存する。`date` フィールドはローカル日付（YYYY-MM-DD）で保存し、タイムゾーン情報は持たない。

### 導出値（キャッシュしない、都度計算する）

- **現在のストリーク**: `sessions` の日付集合から連続日数を算出
- **最長ストリーク**: 同上
- **今週の合計時間**: 今週の `sessions` の `durationMinutes` の合計
- **ヒートマップデータ**: `sessions` → `{ date, count, level }[]` に変換

## エラーハンドリング方針

- **localStorage読み取り失敗（データ破損）**: 「データの読み込みに失敗しました」バナーを表示。設定画面のインポート機能でデータ復旧を案内
- **localStorage書き込み失敗（容量超過）**: 操作失敗をトーストで通知
- **上記以外の予期しないエラー**: コンソール出力のみ

## 技術選定

| 観点           | 選定                              | 理由                                 |
| -------------- | --------------------------------- | ------------------------------------ |
| フレームワーク | **React 19**                      | エコシステムが豊富、情報が多い       |
| ビルドツール   | **Vite**                          | 起動・HMR が高速                     |
| ルーティング   | **react-router-dom (HashRouter)** | サーバー不要でURLが機能              |
| スタイリング   | **Tailwind CSS v4**               | 設定ファイル不要、迅速なUI構築       |
| ヒートマップ   | **react-activity-calendar**       | GitHub風ヒートマップ専用、実装が数行 |
| データ保存     | **localStorage**                  | セットアップ不要、個人用途なら十分   |
| 言語           | **TypeScript**                    | 型安全性                             |
| テスト         | **Vitest**                        | Vite と同一設定、追加設定ほぼゼロ    |

## UI/UX 設計方針

- **最小摩擦で記録完了** を目指す（直前の教材をデフォルト選択）
- 記録保存後、即座にヒートマップとストリークが更新される（視覚的報酬）
- 記録後に短い肯定メッセージをランダム表示（複数パターン。直前と同じメッセージは出さない）
- 肯定メッセージはトーストで数秒表示後に自動消去
- ストリーク断絶後も過去の実績を称え、復帰を肯定的に迎える
- **ネガティブなフィードバックは一切なし**（「警告」ではなく「招待」）
- シンプルで見やすいダッシュボード、過度な統計は避ける
